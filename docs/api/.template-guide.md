# Template Usage Guide

## How to Use This Template

### 1. Copy the Template

```bash
cp docs/api/.template.md docs/api/[layer-dir]/[filename].md
```

### 2. Replace Placeholders

- `[ClassName/InterfaceName]` → Actual class/interface name
- `[package-name]` → Package name (e.g., "primitives")
- `[Layer X - Layer Name]` → Corresponding layer (e.g., "Layer 1 - Primitives")
- `[version]` → Version introduced (e.g., "0.1.0")

### 3. Optional Sections

Remove sections that don't apply:
- If no constructor, remove "Constructor" section
- If no configuration, remove "Configuration" section
- If no common pitfalls, remove that section
- If no performance considerations, remove that section

### 4. Code Examples

- All examples must be executable
- Prefer examples from `examples/` directory
- Include necessary imports
- Add explanatory comments

### 5. Related Links

- Verify all links work
- Use relative paths
- Include at least 2-3 related links

## Quality Checklist

Before considering a document complete:

- [ ] All placeholders replaced
- [ ] At least 2 code examples included
- [ ] All parameters documented
- [ ] All public methods documented
- [ ] At least 2 related links included
- [ ] Valid markdown (no syntax errors)
- [ ] Code blocks specify language (`typescript`)
- [ ] No emojis (per project style guide)

## Naming Conventions

### File Names
- Lowercase with hyphens: `class-name.md`
- README.md for directory overviews

### Headers
- `#` for main title (class/interface name)
- `##` for major sections
- `###` for subsections
- `####` for specific details

### Code Blocks
```typescript
// Always specify language
// Include explanatory comments
// Use descriptive names
```

### Tables
- Align columns for readability
- Use `Yes/No` for Required column
- `-` for "not applicable" in Default column

## Example Usage

### For a Class

```markdown
# Entity

> **Package:** `@stratix/primitives`
> **Layer:** Layer 1 - Primitives
> **Since:** v0.1.0

## Overview

Base class for all domain entities with identity.

**Key Features:**
- Unique identifier using phantom types
- Timestamp tracking (createdAt, updatedAt)
- Equality comparison by ID

## Import

\`\`\`typescript
import { Entity, EntityId } from '@stratix/primitives';
\`\`\`

...
```

### For an Interface

```markdown
# Logger

> **Package:** `@stratix/abstractions`
> **Layer:** Layer 2 - Abstractions
> **Since:** v0.1.0

## Overview

Logging abstraction interface. All logging implementations must implement this interface.

**Key Features:**
- Multiple log levels
- Structured logging support
- Zero runtime overhead

## Import

\`\`\`typescript
import type { Logger } from '@stratix/abstractions';
\`\`\`

...
```

## Tips

1. **Start with README.md** in each directory for an overview
2. **Copy existing docs** as reference for style
3. **Test all examples** to ensure they work
4. **Link liberally** to related documentation
5. **Keep it concise** - users should understand quickly

## Common Mistakes to Avoid

- Including emojis (violates project style)
- Broken relative links
- Code examples without imports
- Missing language specification in code blocks
- Copying placeholder text from template
